Authentication changes your navigation. An unauthenticated visitor sees "Sign in". A logged-in user sees their avatar, maybe their workspace, and a way to sign out. In this post I'll show how to connect **NextAuth.js v5** session data to **Gwan's NavBar and Avatar** components.

## Installation

```bash
npm install next-auth@beta
```

Set up the auth config and route handler as per the NextAuth v5 docs. The key part for us is accessing the session on the client.

## Reading the session

```tsx
// src/app/providers.tsx
"use client";
import { SessionProvider } from "next-auth/react";

export function AuthProvider({ children }: { children: React.ReactNode }) {
  return <SessionProvider>{children}</SessionProvider>;
}
```

## Building an authenticated navbar

Gwan's `NavBar` accepts menu items and renders the nav. We add a user section by composing `Avatar` alongside it:

```tsx
"use client";

import { useSession, signOut } from "next-auth/react";
import { NavBar, Avatar, AVATAR_VARIANT, Ellipsis } from "gwan-design-system";

export default function AppNavbar() {
  const { data: session, status } = useSession();

  const menuItems = [
    { label: "Dashboard", href: "/dashboard" },
    { label: "Projects", href: "/projects" },
    { label: "Settings", href: "/settings" },
  ];

  return (
    <NavBar
      menuItems={menuItems}
      rightSlot={
        status === "loading" ? (
          <div className="w-8 h-8 rounded-full bg-border animate-pulse" />
        ) : session?.user ? (
          <div className="flex items-center gap-3">
            <Avatar
              variant={AVATAR_VARIANT.CIRCLE}
              src={session.user.image ?? undefined}
              fallback={session.user.name?.[0] ?? "U"}
            />
            <div className="hidden md:flex flex-col">
              <Ellipsis
                text={session.user.name ?? ""}
                className="text-sm font-semibold text-foreground max-w-[120px]"
              />
              <span className="text-xs text-muted-fg">{session.user.email}</span>
            </div>
            <button
              onClick={() => signOut({ callbackUrl: "/" })}
              className="text-xs font-semibold tracking-[0.1em] uppercase text-muted-fg hover:text-foreground transition-colors duration-200"
            >
              Sign out
            </button>
          </div>
        ) : (
          <a
            href="/auth/signin"
            className="text-xs font-semibold tracking-[0.1em] uppercase text-muted-fg hover:text-foreground transition-colors duration-200"
          >
            Sign in
          </a>
        )
      }
    />
  );
}
```

## Protecting routes with middleware

```ts
// middleware.ts
import { auth } from "@/auth";

export default auth((req) => {
  const isAuthenticated = !!req.auth;
  const isProtected = req.nextUrl.pathname.startsWith("/dashboard");

  if (isProtected && !isAuthenticated) {
    return Response.redirect(new URL("/auth/signin", req.url));
  }
});

export const config = { matcher: ["/dashboard/:path*", "/settings/:path*"] };
```

## Role-based nav items

Filter menu items based on the user's role from the session:

```tsx
const menuItems = [
  { label: "Dashboard", href: "/dashboard" },
  ...(session?.user?.role === "admin"
    ? [{ label: "Admin", href: "/admin" }]
    : []),
];
```

## Summary

The pattern is: read `useSession()`, map the result to Gwan components (`Avatar` for the face, `Ellipsis` for the name overflow), and pass them into the navbar's right slot. No custom CSS, no bespoke auth UI.
